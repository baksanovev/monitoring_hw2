### Задание 1 

Установите Zabbix Server с веб-интерфейсом.

#### Процесс выполнения
1. Выполняя ДЗ, сверяйтесь с процессом отражённым в записи лекции.
2. Установите PostgreSQL. Для установки достаточна та версия, что есть в системном репозитороии Debian 11.
3. Пользуясь конфигуратором команд с официального сайта, составьте набор команд для установки последней версии Zabbix с поддержкой PostgreSQL и Apache.
4. Выполните все необходимые команды для установки Zabbix Server и Zabbix Web Server.

#### Требования к результатам 
1. Прикрепите в файл README.md скриншот авторизации в админке.
2. Приложите в файл README.md текст использованных команд в GitHub.

### 1. Подготовка системы

```
sudo apt update
sudo apt -y upgrade
sudo apt -y install wget curl gnupg2 lsb-release
```

### 2. Установка PostgreSQL
```
sudo apt -y install postgresql postgresql-contrib
sudo systemctl enable --now postgresql
```

Создание БД и пользователя:
```
sudo -u postgres psql
```
Внутри psql:

```
CREATE DATABASE zabbix
  WITH ENCODING='UTF8'
       LC_COLLATE='C'
       LC_CTYPE='C'
       TEMPLATE=template0;
CREATE USER zabbix WITH ENCRYPTED PASSWORD '123456789';
GRANT ALL PRIVILEGES ON DATABASE zabbix TO zabbix;
```

### 3. Установка Apache и PHP
```
sudo apt -y install apache2
sudo systemctl enable --now apache2

sudo apt -y install php php-pgsql php-gd php-xml php-bcmath php-mbstring php-ldap php-zip php-json
```
### 4. Подключение репозитория Zabbix 6.0 LTS для Debian 11

```
wget https://repo.zabbix.com/zabbix/6.0/debian/pool/main/z/zabbix-release/zabbix-release_6.0-4+debian11_all.deb
sudo dpkg -i zabbix-release_6.0-4+debian11_all.deb
sudo apt update
```
### 5. Установка Zabbix Server + Web (PostgreSQL + Apache)

```
sudo apt -y install zabbix-server-pgsql zabbix-frontend-php zabbix-apache-conf zabbix-sql-scripts
```
### 6. Импорт схемы БД Zabbix в PostgreSQL
```
zcat /usr/share/zabbix-sql-scripts/postgresql/server.sql.gz | sudo -u zabbix psql zabbix
```

### 7. Настройка Zabbix Server
```
sudo nano /etc/zabbix/zabbix_server.conf
```
Основные параметры:

```
DBName=zabbix
DBUser=zabbix
DBPassword=123456789
DBHost=localhost
```

### 8. Запуск и автозапуск сервисов
```
sudo systemctl restart zabbix-server apache2
sudo systemctl enable zabbix-server apache2
```
Проверка:

```
systemctl status zabbix-server
systemctl status apache2
```
### 9. Открыть в браузере:
```
http://93.77.179.255/zabbix
```

### Задание 2 

Установите Zabbix Agent на два хоста.

#### Процесс выполнения
1. Выполняя ДЗ, сверяйтесь с процессом отражённым в записи лекции.
2. Установите Zabbix Agent на 2 вирт.машины, одной из них может быть ваш Zabbix Server.
3. Добавьте Zabbix Server в список разрешенных серверов ваших Zabbix Agentов.
4. Добавьте Zabbix Agentов в раздел Configuration > Hosts вашего Zabbix Servera.
5. Проверьте, что в разделе Latest Data начали появляться данные с добавленных агентов.

#### Требования к результатам
1. Приложите в файл README.md скриншот раздела Configuration > Hosts, где видно, что агенты подключены к серверу
2. Приложите в файл README.md скриншот лога zabbix agent, где видно, что он работает с сервером
3. Приложите в файл README.md скриншот раздела Monitoring > Latest data для обоих хостов, где видны поступающие от агентов данные.
4. Приложите в файл README.md текст использованных команд в GitHub

### 1. Установка Zabbix Agent (agent-zabbix и glpi)
```
sudo apt update
sudo apt -y install wget gnupg2 lsb-release
wget https://repo.zabbix.com/zabbix/6.0/ubuntu/pool/main/z/zabbix-release/zabbix-release_6.0-4+ubuntu22.04_all.deb
sudo dpkg -i zabbix-release_6.0-4+ubuntu22.04_all.deb
sudo apt update
sudo apt -y install zabbix-agent
```

​
### 2. Конфиг агента на agent-zabbix (сам Zabbix Server, IP 10.128.0.24)
```
sudo nano /etc/zabbix/zabbix_agentd.conf
```

```
Server=10.128.0.24
ServerActive=10.128.0.24
Hostname=agent-zabbix
```
### 3. Конфиг агента на glpi (вторая ВМ - IP 10.128.0.31)
```
sudo nano /etc/zabbix/zabbix_agentd.conf
```
```
Server=10.128.0.24
ServerActive=10.128.0.24
Hostname=glpi
```

### 4. Перезапуск и проверка агентов
```
sudo systemctl restart zabbix-agent
sudo systemctl enable zabbix-agent
sudo systemctl status zabbix-agent
sudo tail -n 50 /var/log/zabbix/zabbix_agentd.log
```
